\section{任務與資料}

本研究以 COLIEE 2026 Task~1 為主要評估資料，並以 COLIEE 2025 作為方法探索與失敗分析的輔助資料。Task~1 的目標不是找出一般語義上相似的案例，而是對每個 query case 找出其 noticed cases。若某案例曾被 query case 引用（referenced），則視為該 query 的 noticed case。由於官方已將案例內文中的引用資訊移除（redacted），系統必須根據案件事實、推理與法律語意，自動推斷哪些既有案例最可能支撐該判決。

\subsection{資料來源與切分}

資料來源為加拿大聯邦法院（Federal Court of Canada）判例。訓練資料提供 query cases、整體 candidate pool，以及每個 query 對應的 noticed cases 標註；測試資料只提供 query cases，系統必須自動預測 noticed cases。也就是說，整個流程從查詢、檢索、排序到輸出結果，都必須完全自動化完成。

表~\ref{tab:dataset-stats} 彙整了各年度資料的 query 數量、依年份區分的 candidate pool 規模，以及每個 query 平均 noticed cases 數。COLIEE 2025 提供帶標註的 train 與 valid split；對於 COLIEE 2026，本文使用釋出的帶標註 training set，以及未提供 labels 的官方 test queries。 在目前的實作設定中，COLIEE 2026 的判決書庫共有 7,708 篇文件，並以既有標註再切分為 1,600 個 train queries 與 401 個 valid queries。模型訓練與參數選擇主要根據這個 train/valid split 進行，最終分析則報告 2026 valid 與 2026 train 的 top-5 micro-average F1、precision 與 recall。COLIEE 2025 則主要用來展示訓練失敗、動態負樣本修正，以及 scope filtering 的早期效果。

\subsection{整篇判決書與引用上下文兩種 Query 視角}

本文比較兩種 query 視角。第一種是整篇判決書視角，即以清理後的整篇判決書作為 query 與 candidate。第二種是引用上下文視角，延續 THUIR 的 reference sentence extraction 思路，只保留 citation-redaction placeholders 附近的局部上下文；若擷取結果過短，則再以前處理全文前段補足。本文將它視為一個必須重新驗證的 query 假設，而不是直接接受。為了公平比較，在這組 ablation 中，候選文件一律固定使用整篇判決書視角的 embeddings，僅切換 query 視角。

整篇判決書視角主要保留判決全文正文，並移除 suppression placeholders 的表面字串，例如 \texttt{FRAGMENT\allowbreak\_SUPPRESSED}。此外，也會刪除像是「[translation]」、「[sic]」與「[End of document]」這類編輯性標記，並移除語言偵測結果為法文的段落。若清理後正文仍不含 \texttt{Summary:}，則只有在原始判決書本來就存在可擷取的摘要區塊時，才會把該摘要前置補入。引用上下文視角則先擷取 citation-redaction placeholders 附近的句子與相鄰上下文，再套用與整篇判決書視角相近的清理流程，並在內容過短時以前處理全文前段補長。這表示它並不是被硬截成最多 512 個英文詞，512 只是觸發補長的門檻。以 COLIEE 2026 而言，整篇判決書視角平均為 3,965.52 個英文詞，而引用上下文視角平均為 882.54 個英文詞，中位數為 484。因此，第二種視角並不是純粹的引用片段，而是「以引用上下文為核心、必要時由全文前段補長」的 query 版本。

\subsection{長文本特性與法律領域繼續預訓練資料}

如表~\ref{tab:length-stats} 所示，COLIEE 2026 的清理後整篇判決書文件具有明顯長文本特性，且已有 94.80\% 的文件短於 12,288 tokens，因此以 3 個 4096-token chunks 近似整篇文件是合理設計。

除了 COLIEE 官方資料外，本文亦使用 Hugging Face 上的 Common Pile CaseLaw Access Project subset\footnote{\url{https://huggingface.co/datasets/common-pile/caselaw_access_project_filtered}} 作為法律領域 continued pretraining 語料。該資料包含 6,735,525 篇美國判決書，時間跨度約 365 年。這個語料不直接提供 COLIEE 監督訊號，而是用來在對比式 fine-tuning 前進行 DAPT。受限於單張 RTX 4080 SUPER 16GB 與訓練時間成本，這個 DAPT 階段約訓練 60 小時後，仍只覆蓋 0.384 epoch，因此本文將其視為「有用但尚未飽和」的 domain adaptation 步驟。
