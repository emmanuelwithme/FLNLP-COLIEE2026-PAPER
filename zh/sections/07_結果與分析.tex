\subsection{動態負樣本與年份範圍約束}

依照類似 THUIR 的流程，若先以 BM25 top-100 建立 hard-negative pool，再在每次更新時抽取 15 篇負樣本，模型在 COLIEE 2025 valid 上只得到約 0.00515 的 F1，幾乎等於 random guess 的 0.00451。這說明法律判決檢索中的 negative difficulty 不能只由 lexical rank 定義；真正讓模型開始收斂的關鍵，是改用每個 epoch 依當前模型分數動態抽樣的 hard negatives。當負樣本選取改動後，探索階段的 2025 valid F1 立即提升到 0.12834。

scope filtering 則提供另一層穩定性。它的法律直覺很直接：法官不可能引用未來判決，因此未來案件不應混入負樣本與候選集合。在 COLIEE 2025 valid 上，單純把 ModernBERT fp16 SFT 的訓練資料加上年份範圍約束，F1@5 便由 0.11931 提升到 0.12189。雖然增幅不大，但方向穩定，也與法律引用的時間順序一致。

另一個必須澄清的程式事實是：目前的 scope 不會刪除 gold positives。它只限制 similarity scoring 與負樣本候選池，而正樣本仍直接沿用官方標註。實際稽核目前的 contrastive JSON 後可見，在 2026 train 與 valid 中，仍分別有 256/6,595（3.88\%）與 81/1,654（4.90\%）筆 positive 樣本落在 heuristic year scope 外但被保留進訓練。這樣做是刻意的，因為年份抽取仍是規則式近似；若直接用 scope 篩掉 positives，反而可能誤刪真正的 noticed cases。

\subsection{Learning-to-Rank 與 Post-processing}

learning-to-rank 在本文中不是單純的黑箱 reranker，而是把法律檢索中有意義的訊號結構化後交給 LightGBM 學習。表~\ref{tab:ltr-features} 列出目前使用的全部 28 個特徵，涵蓋 lexical、dense、rank position、長度、placeholder、年份與 chunk aggregation 七類訊號；\texttt{query\_id} 與 \texttt{candidate\_id} 僅作識別欄，不進入訓練。這些特徵的設計重點，是讓 LTR 同時看到全文檢索分數與法律文本的結構訊號。

最後，將本文結果與表~\ref{tab:thuir-table3-excerpt} 對照，可以更清楚看出長文本能力的重要性。根據表~\ref{tab:thuir-table3-excerpt}，不限長度 BM25 的 F1 為 0.1541，而 512-token SAILER 為 0.1385，BM25 高出 0.0156。相較之下，在本文未參與訓練的 COLIEE 2026 valid 上，BM25 的 F1 為 0.1499，而 ModernBERT + DAPT + SFT 可達 0.1650，高出 BM25 0.0151；同一 split 上，SAILER 只有 0.1213，反而比 BM25 低 0.0286。這說明對法律判決檢索而言，可讀取 4096 tokens 的長文本 encoder，已明顯優於只能輸入 512 tokens 的 BERT 類模型。即使目前硬體條件只能完整覆蓋 56.11\% 的判決書長度，長文本模型仍已能超過不限長度的 lexical baseline。
