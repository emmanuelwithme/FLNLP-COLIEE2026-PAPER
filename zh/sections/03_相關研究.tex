\section{相關研究}

本研究主要位於四條研究脈絡的交會處：法律判決檢索、長文本 encoder、對比式 dense retrieval 訓練策略，以及後段 learning-to-rank 融合。

\subsection{法律判決檢索與 COLIEE}

COLIEE Task~1 以判決書對判決書的相關案例檢索為核心，與一般網頁搜尋相比，法律檢索文本更長、語言更正式，且存在大量程序性與引用性內容。THUIR@COLIEE 2023 \cite{li2023thuir} 以 SAILER 為主體，結合 reference sentence extraction、heuristic pre-processing、後處理與 learning-to-rank，取得當年度最佳成績。其方法的重要啟示有二：一是結構知識與引用位置確實對法律文本有幫助；二是 query 應如何表示，可能比模型本身更影響最終結果。本研究延續這條脈絡，但把焦點放在兩個 THUIR 論文沒有完整回答的問題上：reference-centered query 是否仍適合長文本 encoder，以及 dense retrieval 是否主要受限於文本截斷。

\subsection{Lexical Retrieval 與 BM25 基線}

BM25 是資訊檢索中的經典 lexical baseline \cite{robertson2009bm25}。在法律判決檢索任務中，BM25 的重要性特別高，因為判決書中的關鍵法律概念、案例名稱與程序詞彙往往具有強烈的精確匹配訊號。THUIR 的結果與本研究的初步實驗都顯示，當 dense encoder 受到嚴格長度截斷時，BM25 仍可能優於神經式模型。因此，本研究並未將 BM25 視為低門檻對照，而是視為需要被嚴肅超越的強基線。

\subsection{長文本 Encoder 與 ModernBERT}

ModernBERT \cite{warner2024modernbert} 是近年針對 encoder-only Transformer 所提出的現代化改良模型，兼顧較長上下文、較佳的記憶體效率與較快的推論速度。對本研究而言，它最直接的吸引力在於原生支援 8192 tokens，這使其成為檢驗「長文本是否為法律檢索關鍵瓶頸」的合適工具。然而，長上下文能力並不代表可直接遷移至法律檢索任務，特別是當目標資料高度領域化、且 query 與 candidate 皆為長篇判決書時，仍需要 task-specific supervised fine-tuning 與 domain adaptation。本研究的實驗正是沿著這條脈絡展開：先驗證長文本假設，再追問「長文本能力本身」與「任務特化訓練訊號」究竟各自扮演什麼角色。

\subsection{對比式學習與負樣本選取}

Dense retrieval 的對比式學習通常仰賴正樣本與負樣本之間的區分難度來形成有效訊號。若負樣本過於容易，模型難以學到可泛化的判別邊界；但若負樣本來源固定且與目前模型狀態脫節，訓練也可能不穩定。本研究一開始依照類似 THUIR@COLIEE 2023 的流程，先以 BM25 top-100 候選建立 hard-negative pool，再在每個訓練步驟中抽取 15 篇負樣本；但在法律判決檢索設定中幾乎無法收斂。這說明 hard negative 並非只要「看起來困難」即可，還需要與當前模型的混淆情形同步更新。基於此，本研究改以每個 epoch 重新計算 query--candidate 相似度，並依此動態調整負樣本抽樣機率，最終成為本研究最關鍵的方法改動。

\subsection{Learning-to-Rank 與後段融合}

除了 bi-encoder 檢索本身之外，THUIR 在 WSDM Cup 2023、網頁搜尋排序與 NTCIR-16 WWW-4 任務上也持續展示了 learning-to-rank 融合的效果 \cite{chen2023wsdmcup,li2023websearch,yang2023ntcir}。這類方法的共同核心是：不要只依賴單一檢索分數，而是把 lexical、dense、長度、結構與後處理訊號一起交給排序器學習。本研究延續此脈絡，但把特徵重新設計為適合法律判決檢索的版本，例如 placeholder 數量、年份差、以及 1--3 個長文本 chunks 的聚合相似度，並以 query-specific 年份範圍過濾與 cut-off search 作為最終 post-processing。
